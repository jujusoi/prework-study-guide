HTML

  • Elements are types of content found on a webpage. They are composed of an opening tag and a closing tag represented by angle brackets ("< >"). Some can be visible to the user, and others are not. There are four common element types:
    1. Text Elements
    2. Image Elements
    3. Header Elements
    4. List Elements
  • There's also elements used to divide & structure content:
    1. Semantic Elements
    2. Content division Elements
    3. Meta-related Elements
  • The head element contains information about the webpage. This information is known as "metadata", which helps the brower render the webpage. Child elements within the head element are not visible to the user.
  • The body element contains content for the page which can be viewed by the user. Elements we want the user to see and interact with must be placed here.
  • Like head elements, there are also Semantic elements. Semantic elements are children of the body, but are there to properly structure the page. For example.
    1. Header
      2. The header is the first step in body element organisation. It appears at the top of the page, and usually contains a navigation menu and introductory content.
    2. Main
      3. The main is the main area of content for the body. It provides the user with chunks of information and resources.
    3. Footer
      4. The footer appears at the bottom of a apge, usually containing information such as author, copyright, contacts, sitemaps and further navigation.

CSS

  • A programming language used to define how webpage content is presented to users (formatting html with stylistic rules and options)
  • There are 3 types of CSS styling options:
    1. Inline CSS
      2. Used to style an element directly in the html file (inside an element tag).
    2. Internal CSS
      3. Used to style elements within the head element of a html file (inside the head, style child element)
    3. External CSS
      4. Separate, linked CSS files referenced in the head of the html page which allows devs to remotely edit CSS rules in another document.
  • In a declaration block, start off with the selector, followed by curved brackets, with a property:value; end bracket. Like such: div {color:blue; text-align: justify;}
  • There are lots of properties in CSS, however some basic structural properties include margin, padding, text-align, and display.
    • Margin: Indicates how much space is wanted outside of an element
    • Padding: Indicates how much space is wanted inside the element
    • Text-align: A way to center or displace an element on an axis
    • Display: Displays content on the page differently depending on the value attributed.
  • You can also attribute CSS rules to a whole bunch of elements under the same code using class selectors. Attributing a class to a html element will allow you to use it as a CSS selector via .classname

Git

  • Git is an open-source version control system which allows devs to collaborate, edit, upload and share work to coworkers. It can also be used to navigate around your local system using an array of commands.
  • Some normal navigation commands include
    1. pwd - shows you your current directory
    2. cd - lets you change directories
    3. cd .. - lets you move back one directory
    4. ls - shows you all files in the current directory
    5. mkdir - creates a new directory
    6. touch - creates a new file in current directory
    7. start - runs a file
    8. rm - removes a file
    9. rm -r - removes a directory
  • During collaboration, developers will have to utilize "Branches", which are clones of main branch codebases which allow further editing. These can also be accessed via commands.
    1. git - precedes every git related command
    2. checkout - moves working branch to the branch specified
    3. -b - creates new branch
    4. feature/develop - state the type of branch
    5. status - tells you what branch you're on
    6. add - adds branch modifications
    7. -A - specifies to add all modifications, or add all __
    8. pull - pulls branch into local environment
    9. origin - specifies the source of the branch (repository) via the directory that is linked to GitHub
    10. main - specifies name of branch you want to pull
